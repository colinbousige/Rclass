Reproducible data treatment with R

4 Simple variables

In R, the variable attribution is done through the arrow operator <- instead of the = one – for historical reasons. The equal sign, =, would still work though, but you should just take up the habit of using <-.

In reality, scalar numbers don’t exist in R, they are simply 1 element vector: this is why [1] is printed in the following outputs.

To understand what you will encounter further down this class, inputs (what you type in the R script and hit Ctrl+⏎ (⌘+⏎)) are the shaded areas with syntax coloring, while outputs (what R returns) are the shaded are underneath starting with ## [1]. In case the output spans several lines, each line would start with ## [x] where x is the index of the first element printed on the line.

4.1 Scalars and booleans

Separate in-line instructions with a ; or just skip a line. Different ways of defining a scalar:

x <- 1
x
## [1] 1
y <- floor(1.2); y
## [1] 1

You can use the R console as an advanced calculator:

1+3
## [1] 4
x-y
## [1] 0

4.1.1 Booleans

Booleans are handled with the TRUE and FALSE keywords. Any number non 0 is equivalent to TRUE, 0 is FALSE.

z <- TRUE; typeof(z)
## [1] "logical"
as.logical(1)
## [1] TRUE
as.logical(0)
## [1] FALSE
x <- 1; y <- 2
x == y # is equal
## [1] FALSE
x != y # is not equal
## [1] TRUE
x < y  # is smaller than
## [1] TRUE
x <= y # is smaller or equal than
## [1] TRUE
x == y & x < y  # operator "and"
## [1] FALSE
x == y | x < y  # operator "or"
## [1] TRUE

4.1.2 Special values

R handles infinity, NaN, and has \(\pi\) defined. Missing numbers are handled through the NA keyword (the function is.na() checks if the given value contains NA).

pi
## [1] 3.141593
10/0
## [1] Inf
-10/0
## [1] -Inf
0/0
## [1] NaN
is.na(NA)
## [1] TRUE

4.1.3 Complex values

R also natively handles complex values:

1+i   # not a valid complex value
## Error in eval(expr, envir, enclos): object 'i' not found
1+1i  # valid complex value
## [1] 1+1i
exp(1i*pi)
## [1] -1+0i
sqrt(-1)
## Warning in sqrt(-1): NaNs produced
## [1] NaN
sqrt(-1 + 0i)
## [1] 0+1i
Im(exp(1i*pi))
## [1] 1.224647e-16

4.2 Strings

A string is defined between quotation marks "string". Here are some operations on strings :

phrase <- " Hello World " # definition of a string
paste("phrase =", phrase, ", class(phrase) =", class(phrase))
## [1] "phrase =  Hello World  , class(phrase) = character"
substr(phrase, 2, 5) # sub-string
## [1] "Hell"
paste(tolower(phrase), toupper(phrase), sep=" - ") # change case
## [1] " hello world  -  HELLO WORLD "
sub("o", "a", phrase)  # change the first occurrence of "o" in "a"
## [1] " Hella World "
gsub("o", "a", phrase) # change all occurrences of "o" in "a"
## [1] " Hella Warld "
trimws(phrase)         # trim white spaces
## [1] "Hello World"
phrase2 <- "1234"
phrase2 - 4321             # won't work: string - double
## Error in phrase2 - 4321: non-numeric argument to binary operator
as.numeric(phrase2) - 4321 # conversion of string to double
## [1] -3087

For more complex operations, see the stringr package or the string cheatsheet.

x <- "100"; y <- 99.6; 
x - ceiling(y)
Show output

1 / sin(as.numeric(x) - ceiling(y))
Show output

cos(pi/2)==cos(3*pi/2)

cos(pi/2)-cos(3*pi/2)

all.equal(cos(pi/2),cos(3*pi/2))
